2. ¿Qué estamos construyendo?

Una app móvil de reporte ciudadano ambiental. Cualquier persona puede fotografiar un problema ambiental (basura, escombros, aguas contaminadas), y la app lo geolocaliza automáticamente y lo publica en un mapa compartido.

El diferencial es el mapa de calor: las zonas con más reportes se muestran en rojo, permitiendo a autoridades o ciudadanos identificar los sectores más afectados.

Arquitectura del Sistema

El proyecto tiene dos partes completamente independientes que se comunican por HTTP:

📱 Usuario
   ↕ abre la app
React Native (Expo) — frontend, corre en el celular
   ↕ HTTP requests (JSON) al puerto 8000
Nginx — servidor web, recibe las peticiones HTTP
   ↕ FastCGI al puerto 9000
PHP-FPM (Laravel 12) — ejecuta la lógica de la API
   ↕ TCP al puerto 3306
MySQL 8 — guarda usuarios, reportes y categorías
💡
Punto clave: El backend y el frontend nunca se "mezclan". El backend solo devuelve JSON. El frontend solo muestra lo que recibe. El contrato entre ambos está definido en docs/API.md.

Cómo funciona la autenticación

Usamos Laravel Sanctum con tokens. El flujo es:

App ──→ POST /api/login { email, password }
Backend verifica credenciales → genera token
App ←── { token: "5|abc123xyz..." }
────────────────────────────────────────
App ──→ GET /api/reports + header: Authorization: Bearer 5|abc123...
Backend valida el token → sabe quién eres → devuelve datos

3. Setup Backend — Bastian, Mathias, Sebastian

📋
Requisito único: Docker Desktop instalado y corriendo. No necesitas instalar PHP, Composer ni MySQL en tu máquina. Todo corre dentro de los contenedores.

¿Qué hace Docker por nosotros?

Docker crea 4 "computadores virtuales" (contenedores) en tu máquina, todos conectados entre sí:

ContenedorQué esPuerto en tu máquina
reportes_appPHP 8.2 — ejecuta LaravelInterno (9000)
reportes_nginxServidor web — recibe tus peticiones8000
reportes_dbMySQL 8 — base de datos3306
reportes_phpmyadminPanel visual para gestionar la BD8080

Primera vez — Paso a Paso

  1. 1
    Clonar el repositorio 
    git clone https://github.com/Bazzty/proyecto-reportes.git
cd proyecto-reportes && git checkout develop
  2. 2
    Crear tu archivo de entorno con tus contraseñas 
    cp backend/.env.example backend/.env
# Abre backend/.env y completa estas líneas:
DB_PASSWORD=tu_contraseña_aqui
DB_ROOT_PASSWORD=otra_contraseña_aqui

    Puedes poner cualquier contraseña, es solo para tu entorno local.

  3. 3
    Construir y levantar los contenedores 
    cd backend
docker compose up -d --build
# La primera vez descarga imágenes, puede tardar 3-5 minutos
  4. 4
    Configurar Laravel (solo una vez) 
    docker compose exec app php artisan key:generate
docker compose exec app php artisan vendor:publish --provider="Laravel\Sanctum\SanctumServiceProvider"
docker compose exec app php artisan migrate
  5. 5
    Verificar que todo funciona

    Abre tu navegador en http://localhost:8000/up — debe mostrar un mensaje verde.

    Abre http://localhost:8080 — debe mostrar phpMyAdmin.

Uso Diario

# Encender (cada vez que quieras trabajar)
cd backend && docker compose up -d

# Apagar (cuando termines)
docker compose down

# Ver logs si algo falla
docker compose logs app
docker compose logs db

Comandos de Laravel más usados

# Crear una migración nueva (tabla nueva)
docker compose exec app php artisan make:migration create_reports_table

# Crear modelo + migración + controlador en un paso
docker compose exec app php artisan make:model Report -mc

# Correr migraciones pendientes
docker compose exec app php artisan migrate

# Borrar todo y recrear desde cero (útil en desarrollo)
docker compose exec app php artisan migrate:fresh

# Correr los tests
docker compose exec app php artisan test

# Abrir la consola interactiva de Laravel
docker compose exec app php artisan tinker

5. Flujo de Trabajo con Git

🚫
Regla de oro: Nadie sube código directamente a main ni a develop. Siempre creas tu rama feat/, trabajas ahí, y abres un Pull Request para que Bastian lo revise.

Estructura de Ramas

main ────────────────────────────── código estable para la entrega final
      ↑ solo via Pull Request desde develop (aprobado por Bastian)
develop ──────────────────────────── integración del equipo
         ↑ solo via Pull Request desde feat/*
feat/login-api   feat/pantalla-login   feat/mapa-heatmap

Ciclo Completo por Tarea

  1. 1
    Siempre partir desde develop actualizado 
    git checkout develop
git pull origin develop
  2. 2
    Crear tu rama con el nombre de la tarea 
    # Formato: feat/descripcion-corta
git checkout -b feat/login-api           # backend
git checkout -b feat/pantalla-login      # frontend
  3. 3
    Trabajar y hacer commits frecuentes 
    git add .
git commit -m "feat: endpoint POST /api/login con Sanctum"

    Haz un commit cada vez que algo funciona, no esperes a terminar todo.

  4. 4
    Subir tu rama a GitHub 
    git push origin feat/login-api
  5. 5
    Abrir un Pull Request en GitHub

    Ve a GitHub → tu repo → aparece un banner "Compare & pull request" → haz click → asegúrate de que el PR va hacia develop (no main) → asigna a Bastian como reviewer → Submit.

  6. 6
    Esperar revisión y hacer merge

    Bastian revisa el código, puede pedir cambios o aprobarlo. Una vez aprobado, hace merge y elimina la rama. ¡Tarea completada!

Formato de Commits

PrefijoCuándo usarloEjemplo real

fix:Corrección de un bugfix: GPS no pedía permisos en Android
docs:Solo documentacióndocs: agregar ejemplos JSON al API.md

Relaciones entre tablas

users ──hasMany──▶ reports  (un usuario puede tener muchos reportes)
reports ──belongsTo──▶ users  (cada reporte pertenece a un usuario)
categories ──hasMany──▶ reports  (una categoría tiene muchos reportes)
reports ──belongsTo──▶ categories  (cada reporte tiene una categoría)

Cómo se escribe en Laravel (Eloquent)

// Modelo Report
class Report extends Model {
  protected $fillable = [
    'user_id', 'description',
    'latitude', 'longitude',
    'photo_url', 'status'
  ];

  public function user() {
    return $this->belongsTo(User::class);
  }
}
// Crear un reporte en el controlador
$report = Report::create([
  'user_id'     => auth()->id(),

10. Errores Comunes y Soluciones

Docker: "No se estableció la conexión" en phpMyAdmin
MySQL todavía está inicializándose. Espera 30 segundos y recarga la página. Si persiste, el volumen tiene credenciales viejas: 
docker compose down -v   # elimina el volumen con datos viejos
docker compose up -d     # crea todo de nuevo
Laravel: "No application encryption key has been specified"
El archivo .env no tiene APP_KEY. Genera una: 
docker compose exec app php artisan key:generate
Laravel: "SQLSTATE: Connection refused" al hacer migrate
MySQL aún no está listo cuando se intenta conectar. Espera unos segundos y vuelve a intentar. Si el error persiste, verifica que DB_HOST=db en el .env (no 127.0.0.1).
Git: "fatal: The current branch has no upstream branch"
La rama no existe en GitHub todavía. Usa el flag -u la primera vez: 
git push -u origin feat/nombre-de-tu-rama
Expo: "Network request failed" al llamar al backend
El celular no puede conectarse a localhost porque eso apunta al celular mismo, no a tu computador. Cambia la URL en services/api.js por la IP local de tu máquina: 
# En Mac/Linux:  ifconfig | grep "inet "
# En Windows:    ipconfig | grep "IPv4"
# Luego en api.js:
baseURL: 'http://192.168.1.X:8000/api'  ← reemplaza con tu IP
Laravel: "419 CSRF token mismatch"
Estás usando rutas web en vez de rutas API. Asegúrate de que todos tus endpoints estén en routes/api.php y que el frontend llame a /api/... (con el prefijo /api).
Expo: El QR aparece pero el celular no carga la app
Tu celular y tu computador deben estar en la misma red WiFi. Si estás en la universidad y estás en redes separadas (WiFi del cel vs cable del PC), no va a funcionar. Conecta ambos a la misma red.

11. Preguntas Frecuentes

¿Dónde escribo mi código?
Backend: controladores en backend/app/Http/Controllers/, migraciones en backend/database/migrations/, modelos en backend/app/Models/.
Frontend: pantallas en frontend/src/screens/, componentes en frontend/src/components/, llamadas al backend en frontend/src/services/api.js.
¿Por qué no subo vendor/ ni node_modules/?
Porque pesan 100-500 MB y no hace falta. Esas carpetas contienen las librerías descargadas, y cada developer las genera en su máquina corriendo composer install (backend) o npm install (frontend). El archivo composer.json y package.json tienen la lista de qué instalar.
¿El archivo .env va al repositorio?
No. Cada developer tiene su propio .env con sus contraseñas locales. Está listado en .gitignore así que Git lo ignora automáticamente. Solo el .env.example (sin contraseñas) va al repo, para que cada developer sepa qué variables configurar.
¿Puedo subir directo a main o a develop?
No. La rama main está protegida en GitHub: requiere aprobación de Bastian para hacer merge. Siempre creas una rama feat/, trabajas ahí, abres un Pull Request hacia develop y esperas la revisión.
¿Qué hago si hay conflictos en Git?
Avísale a Bastian antes de intentar resolverlos solo. Los conflictos mal resueltos pueden eliminar código de tus compañeros. Para prevenirlos, haz git pull origin develop frecuentemente en tu rama.
¿Cómo pruebo los endpoints del backend sin el frontend?
Usa Postman o Thunder Client (extensión de VS Code). Son herramientas que te permiten hacer peticiones HTTP directamente, como si fueras el frontend. Es la forma estándar de probar APIs.
¿Cómo sé que mi código funciona antes de hacer el PR?
Backend: corre docker compose exec app php artisan test. Frontend: prueba manualmente en tu celular con Expo Go. Asegúrate de que lo básico funciona: no hay errores en consola y la pantalla muestra lo esperado.
¿Qué es un Pull Request?
Es una solicitud en GitHub para fusionar tu rama con develop. Sirve para que Bastian revise tu código antes de integrarlo. Puedes escribir una descripción explicando qué hiciste. Es la forma profesional de colaborar en equipo.

12. Glosario

Términos técnicos que aparecen en el proyecto, explicados en simple.

API (REST)
Sistema de comunicación entre apps. El backend expone "rutas" y el frontend las llama enviando/recibiendo JSON.
JWT / Token
Cadena de texto que identifica a un usuario. Se obtiene al login y se envía en cada petición para demostrar quién eres.
Laravel Sanctum
Paquete oficial de Laravel para autenticación de APIs. Genera y valida tokens de usuario.
Migración
Archivo PHP que describe cómo crear o modificar una tabla en la base de datos. Se corre con php artisan migrate.
Eloquent ORM
Sistema de Laravel para interactuar con la BD usando objetos PHP en vez de SQL crudo. Cada tabla tiene un Modelo.
Docker / Contenedor
Tecnología que empaqueta una aplicación con todo lo que necesita para correr. Permite que todos trabajen con el mismo entorno.
Nginx
Servidor web que recibe las peticiones HTTP y las redirige al servidor PHP. Es la "puerta de entrada" de la API.
FastCGI
Protocolo interno que usa Nginx para comunicarse con PHP. No lo configuras directamente, solo existe en el docker-compose.
Expo / Expo Go
Herramienta que simplifica el desarrollo de React Native. Expo Go es la app del celular que ejecuta tu código sin compilar.
Axios
Librería JavaScript para hacer peticiones HTTP. Es lo que el frontend usa para llamar al backend.
useState / useEffect
Hooks de React. useState guarda datos en la pantalla. useEffect ejecuta código cuando la pantalla carga o cuando algo cambia.
Pull Request (PR)
Solicitud en GitHub para fusionar una rama. Permite revisar el código antes de integrarlo al proyecto.
Heatmap
Mapa de calor. Visualización que colorea zonas según la densidad de datos. Rojo = muchos reportes. Verde = pocos.
multipart/form-data
Formato para enviar archivos (fotos) por HTTP. Es diferente a JSON y se usa cuando hay imágenes en el request.
AsyncStorage
Almacenamiento local en el celular. Se usa para guardar el token de sesión, igual que las cookies en un navegador web.
.env
Archivo de variables de entorno. Guarda configuraciones sensibles (contraseñas, URLs) fuera del código. No va al repositorio.
